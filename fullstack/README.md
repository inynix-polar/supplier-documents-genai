# Тестовое задание — Python/TypeScript (React) Middle Full-stack Engineer

**Тайм-бокс:** ~60 минут · **Формат:** vibe-coding (LLM-ассистент разрешён)

> Самодостаточная заготовка. В сервисе значения из текста документов вытаскиваются
> **regex-паттернами**, хранящимися в YAML-конфигах атрибутов. Составлять и отлаживать
> эти regex вручную — боль аналитика; делаем инструмент.

## Задача — инструмент «Regex Lab»

Страница подбора регулярного выражения по примеру: пользователь вставляет текст,
отмечает **что нужно вытащить (позитивный пример)** и **что нельзя (анти-пример)** —
LLM генерирует regex, UI сразу подсвечивает совпадения.

### Frontend (React + TypeScript) — основное

Каркас отсутствует намеренно — поднимаешь сам (Vite, см. `frontend/README.md`).
Состав страницы и требования — там же. Ключевое: **живая подсветка** совпадений и
анти-примеров на клиенте + экспорт YAML-блока атрибута.

### Backend (FastAPI) — поддерживающее

`POST /api/regex/suggest` в `backend/app/main.py` сейчас заглушка (TODO). Нужно:
- собрать промпт из `text/positives/negatives`, вызвать `LLMClient`;
- безопасно распарсить JSON (клиент намеренно заворачивает ответ в ` ```json `);
- **провалидировать** regex на сервере: компилируется? покрывает позитивы? не задевает
  негативы? При провале — осмысленная ошибка.

`LLMClient` (`backend/app/utils/llm_client.py`) работает с `LLM_FAKE=1` без ключа,
так что UI и backend можно разрабатывать офлайн.

## Запуск

```bash
# backend
cd backend && pip install -r requirements.txt && LLM_FAKE=1 uvicorn app.main:app --reload
# frontend (в другом терминале)
cd frontend && npm create vite@latest . -- --template react-ts && npm i && npm run dev
```

## Критерии приёмки

- [ ] React+TS-приложение собирается, есть подсветка совпадений в реальном времени.
- [ ] `/api/regex/suggest` генерирует regex через `LLMClient` и парсит JSON устойчиво.
- [ ] Сервер валидирует regex по позитивам/негативам перед ответом.
- [ ] Кнопка экспорта выдаёт валидный YAML-блок атрибута.
- [ ] Обработка ошибок: невалидный regex от LLM, пустой ввод, недоступность LLM.
- [ ] Git-история с conventional commits + заполненный `NOTES.md`.
