# Тестовое задание — Middle ML Engineer

**Тайм-бокс:** ~60 минут · **Формат:** vibe-coding (LLM-ассистент разрешён)

> Самодостаточная заготовка модуля сопоставления атрибутов (требование Заказчика
> vs предложение Поставщика). Внешняя инфраструктура не нужна.

## Контекст

Матчинг атрибутов использует стратегии сравнения; текущая `fuzzy` (`rapidfuzz`,
см. `app/modules/fuzzy_matcher.py`) считает **посимвольную** близость и плохо ловит
**семантику**. Пример: «нержавеющая сталь» vs «коррозионностойкая сталь 12Х18Н10Т» —
по символам далеко, по смыслу матч. Из-за этого часть кейсов без нужды улетает в
дорогой LLM-fallback.

## Задача

### 1. Эмбеддинг-матчер

Реализовать `EmbeddingMatcher` (`app/modules/embedding_matcher.py`, заглушка готова),
соблюдая контракт `IAttributeMatcher` (`app/core/interfaces.py`):
- косинусная близость эмбеддингов нормализованных значений;
- модель — лёгкая локальная (`sentence-transformers`, мультиязычная, напр.
  `paraphrase-multilingual-MiniLM`) или любой embeddings-API;
- **кэш эмбеддингов** (одинаковые строки не пересчитываем);
- порог решения (matched/partial/mismatch) — параметр, не хардкод.

### 2. Offline-эвал

`eval/dataset.jsonl` содержит 10 стартовых кейсов — **расширь до ≥20** (синонимы,
единицы измерения, шум; материал — `configs/synonyms.yaml`). В `eval/run_eval.py`
(скелет готов) посчитай **precision/recall/F1** и сравни: `fuzzy` vs `embedding` vs
`fuzzy→embedding fallback`. Вывод — таблица метрик + где какой подход выигрывает.

## Запуск

```bash
pip install -r requirements.txt
python -m eval.run_eval          # пока гоняет только baseline
PYTHONPATH=. pytest -q
```

## Критерии приёмки

- [ ] `EmbeddingMatcher` соответствует контракту, порог конфигурируем.
- [ ] Эмбеддинги кэшируются.
- [ ] Датасет ≥20 кейсов + рабочий скрипт-эвал.
- [ ] precision/recall/F1, сравнение fuzzy vs embedding с выводами.
- [ ] Тест на матчер (happy-path + 1 пограничный кейс).
- [ ] Git-история с conventional commits + заполненный `NOTES.md`.

## Что оцениваем дополнительно (опиши в `NOTES.md`)

- Честность эвала: разбиение, отсутствие утечки, репрезентативность.
- Latency/стоимость: эмбеддинги локально vs LLM-fallback — что выигрываем.
- Как выбрал порог (не подгонял ли под тест) и как калибровать на проде.
